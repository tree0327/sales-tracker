# 부부 가계부 (Couple Household Ledger) — Phase 1 설계

기존 "미용실 매출 관리" 단일 앱을 **부부 공용 가계부**로 전환한다. 남편 급여 · 아내 미용실
매출 · 공동/개인 지출을 하나의 투명한 가계부로 기록하고, 매달 고정지출부터 빼고 시작해
중앙 화면에서 실시간 잔액을 본다. 프로토타입(검증 완료)을 실제 React 앱으로 구현한다.

## 핵심 모델

```
이번 달 잔액 = − 고정지출 + (매출·급여) − 변동지출
```

- 매달 **고정지출**로 `−` 시작 → **매출/급여** 들어오면 `+` → **지출**하면 `−`.
- 모든 데이터는 부부가 **투명하게 공유**(전부 열람·수정 가능). "누가"는 태그(owner)로 구분.

## 데이터 모델 (Supabase)

### transactions (수입·지출 일반화 — 기존 sales_records 대체)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | bigint identity PK | |
| flow | text | 'income' \| 'expense' |
| amount | numeric | 원금 |
| final | numeric | 실수령(매출 카드 = floor(원금×0.9)), 그 외 amount와 동일 |
| category | text | 지출: 식비/카페 등 · 수입: 매출/급여/기타수입 |
| owner | text | 'wife' \| 'husband' \| 'joint' |
| method | text | '카드' \| '현금' \| '계좌' |
| memo | text | |
| date | timestamptz | 거래일 |
| created_at | timestamptz | |

### fixed_expenses (고정지출 — 매달 자동 `−` 시작점)
id, name, amount, day(결제일), method, created_at

### expense_categories (지출 카테고리 — 설정에서 추가/삭제)
id, name(unique), sort, created_at · 기본값 시드

RLS: **로그인(authenticated) 사용자만 전체 접근**, anon 차단. (부부 2인 전용)

### 마이그레이션
기존 `sales_records` → `transactions`(flow=income, category='매출', owner='wife',
method=현금/카드, final 보존)로 1회 이관(transactions가 비어있을 때만).

## 사용자 · 인증

- Supabase 이메일/비번 로그인(가입 경로 없음, 계정은 대시보드에서 생성).
- 로그인 사용자 → `wife`/`husband` 매핑: `user.user_metadata.role` 우선, 없으면
  `src/lib/members.js`의 이메일 매핑, 그래도 없으면 기본값.
- 입력 시 owner가 **로그인한 사람으로 자동 선택**되고, 직접 탭해야만 변경. (매출=아내,
  급여=남편은 성격상 고정)

## 화면 (하단 탭: 홈 · 지출 · ＋ · 매출·급여 · 기록, 우상단 ⚙ 설정)

1. **홈** — 현재 잔액(랜딩) + 흐름 계산(−고정 / +수입 / −변동 / =잔액) + 대메뉴 카드
2. **지출관리** — 서브탭 `고정 · 아내 · 남편 · 공금`. 고정=목록/추가, 사람=카테고리별 사용 막대 + 내역
3. **매출/급여관리** — `매출관리(아내·현금/카드+수수료)` / `급여관리(남편·금액만)`
4. **기록** — 월 이동 + 필터(종류 · 소유자 · 결제수단) + 월별/일별 보기
5. **설정** — 지출 카테고리 추가/삭제, 계정/로그아웃

## 입력 시트 (커스텀 숫자 키패드 — 시스템 키보드 미사용)

지출/수입 토글 → 금액(키패드+빠른금액) → 카테고리 → 누구(자동선택) → 결제수단 → 날짜 → 메모.
- **급여**: 결제수단 숨김(금액만, method='계좌').
- **매출**: 아내 고정 · 현금/카드 · 카드는 10% 수수료로 실수령 계산.

## 기술 스택
